Делаем на минималках.
После того, как разберусь с настройкой проекта, набросаем следующие важные части:
1. Хранение и отображение списка.
2. Добавление новых элементов.
3. Удаление элементов.

Необязательным можно сделать ещё хранение выполненых задач и выдача. Да, сделаем.
Содержание страницы будет условно разделено на два блока.
1. Список актуальных задач. Основной функционал, включающий:
1.1. Добавление новых элементов.
1.2. Удаление элементов из списка (напротив каждого элемента кнопка).
2. Список удалённых задач.


Можно приступать.

Приложения

На сайте будет всего одна страничка, на которой будет размещён весь функционал.

В бд будут храниться все задачи, вне зависимости от того, выполнена или нет, они будут одного типа сущности.
Различие будет в одном булевом параметре.

Итак, сущность Task:
id - численный идентификатор (реализовано по умолчанию)
taskname - текст (имя) задачи
status - актуальная или выполненная, булево значение, чтобы не заморачиваться при создании по умолчанию поле будет заполнено значением True
